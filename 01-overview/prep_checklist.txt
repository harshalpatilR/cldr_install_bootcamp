5. Pick a cluster console administration tool
> • The main goal for this tool is to allow running commands on many nodes of a cluster in parallel.
> • There are many alternatives available and it's down to personal preference. You can pick and choose:
> - csshX - it’s lightweight and sufficient for managing a small number of nodes - https://github.com/brockgr/csshx
> - iTerm2 - it supports multiple windows and a broadcast input feature - http://iterm2.com
> - pssh - parallel ssh - https://www.cyberciti.biz/cloud-computing/how-to-use-pssh-parallel-ssh-program-on-linux-unix/
